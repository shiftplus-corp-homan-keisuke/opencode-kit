# コマンド一覧

合計: 12

| コマンド | 説明 |
| --- | --- |
| /brainstorm | 構造化ブレインストーミングを行う。複数案の提示、比較、推奨までを整理する（コードは書かない）。 |
| /create | 新規アプリ作成の対話型ワークフロー。要件確認→構成決定→実装→プレビューまで進める。 |
| /debug | 体系的な問題調査。症状整理→仮説→検証→修正と再発防止までを一連で扱う。 |
| /deploy | デプロイの安全手順を実行する。事前チェック（型/ lint/テスト/セキュリティ/ビルド）とプラットフォーム検出を含む。 |
| /docs | ドキュメント生成と更新を自動化する。generate/update/serve/config init を扱い、出力確認まで行う。 |
| /enhance | 既存アプリの機能追加・改善。現状分析→変更計画→実装→検証/プレビューを行う。 |
| /orchestrate | 複数エージェントでの協調作業を行う。最低3エージェント、計画→承認→実装の2フェーズを強制する。 |
| /plan | 計画作成専用。コードを書かず、質問で要件を整理して計画ファイルとタスクリストを作る。出力は `./specs/{slug}/{slug}-plan.md` と `./specs/{slug}/{slug}-task.md`。 |
| /preview | プレビューサーバーの起動/停止/再起動/状態確認/ヘルスチェックを行う。フレームワークを検出して適切に起動する。 |
| /status | プロジェクトとエージェントの状態を一覧化する。構成検出、ファイル統計、プレビュー状態をレポートする。 |
| /test | テストの実行/生成/カバレッジ表示を行う。フレームワーク検出に基づいて適切に実行する。 |
| /ui-ux-pro-max | 大規模なUI/UXデザイン指針の生成。デザインシステム抽出とスタック別ガイドを提供する。 |
